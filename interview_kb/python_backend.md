# Python 后端开发工程师 — 面试题库

## 一、Python 语言基础

### Q1: Python 的 GIL 是什么？对多线程有什么影响？
**参考答案**：
GIL（Global Interpreter Lock，全局解释器锁）是 CPython 中的一个互斥锁，确保同一时刻只有一个线程执行 Python 字节码。
- 影响：CPU 密集型任务多线程反而更慢（线程切换开销 + 锁竞争）
- 解决方案：多进程（multiprocessing）、异步编程（asyncio）、使用 C 扩展释放 GIL
- Python 3.13 引入了 Free-threaded 模式（实验性），允许禁用 GIL

**评分标准**：
- 9-10分：能解释 GIL 存在的历史原因（引用计数 GC），能对比多线程/多进程/协程的适用场景，了解 Python 3.13 的去 GIL 进展
- 6-8分：能说清楚 GIL 是什么，知道多进程是解决方案
- 1-5分：只知道 GIL 让多线程变慢，不知道原因

### Q2: 装饰器的原理是什么？写一个带参数的装饰器。
**参考答案**：
装饰器本质是一个接受函数作为参数并返回新函数的高阶函数。`@decorator` 语法糖等价于 `func = decorator(func)`。
带参数装饰器需要三层嵌套：外层接收参数 → 中层接收函数 → 内层执行逻辑。
常用场景：日志记录、权限校验、缓存、重试机制。

**评分标准**：
- 9-10分：能解释闭包原理，能写带参数装饰器，了解 `functools.wraps` 保持元信息
- 6-8分：能写简单装饰器，理解基本用法
- 1-5分：知道 @ 语法但不会手写

### Q3: 生成器和迭代器的区别？yield 的工作原理？
**参考答案**：
- 迭代器：实现 `__iter__` 和 `__next__` 方法的对象
- 生成器：使用 yield 关键字的函数，是创建迭代器的简便方式
- yield 原理：函数执行到 yield 时暂停并保存状态，下次调用 next() 时从暂停处恢复
- 优势：惰性计算，节省内存（处理大文件/大数据流时很重要）

**评分标准**：
- 9-10分：能对比生成器表达式 vs 列表推导式的内存差异，了解 send()/throw()/close() 方法，了解 yield from
- 6-8分：知道生成器是特殊的迭代器，知道 yield 能节省内存
- 1-5分：用过但讲不清原理

### Q4: Python 的内存管理机制？
**参考答案**：
- 引用计数为主 + 标记清除（循环引用）+ 分代回收
- 小整数对象池（-5 到 256 的整数被缓存）
- 字符串 intern 机制
- `__del__` 析构函数的注意事项
- 上下文管理器（with 语句）确保资源释放

## 二、Web 框架（FastAPI / Flask / Django）

### Q5: FastAPI 的依赖注入系统是怎么工作的？
**参考答案**：
FastAPI 的 `Depends()` 使用 Python 的类型注解和参数检查来实现依赖注入。
- 通过 inspect 模块分析函数签名
- 支持嵌套依赖（依赖可以有自己的依赖）
- 自动处理缓存（同一请求内相同依赖只执行一次）
- 支持 yield 依赖（用于数据库连接等需要清理的场景）

### Q6: 中间件（Middleware）的实现原理？你用中间件解决过什么问题？
**参考答案**：
中间件是一个在请求-响应周期中拦截请求/响应的组件。
- ASGI 中间件：接收 scope + receive + send，包装 send 函数
- 常用场景：CORS 处理、请求日志、认证鉴权、请求限流、异常处理
- FastAPI 中使用 `@app.middleware("http")` 装饰器

## 三、数据库

### Q7: SQLAlchemy ORM 的 N+1 查询问题是什么？如何解决？
**参考答案**：
N+1 问题：查询主表 1 次 + 每条记录查关联表 1 次 = 共 N+1 次查询。
解决方案：
- `joinedload()`：LEFT JOIN 一次性加载关联数据
- `selectinload()`：IN 查询分两次加载（适合一对多）
- `lazyload()`：需要时再加载（默认行为）
- 批量查询优化：合理使用索引、避免在循环中查询

### Q8: 数据库索引的底层数据结构？B+ 树 vs Hash 索引？
**参考答案**：
- B+ 树：所有数据存储在叶子节点，非叶子节点只存索引键；叶子节点形成有序链表支持范围查询
- Hash 索引：等值查询 O(1)，但不支持范围查询和排序
- MySQL InnoDB 使用 B+ 树，主键索引叶子节点存完整行数据（聚簇索引）

## 四、系统设计

### Q9: 如何设计一个高并发的 RESTful API？
**参考答案**：
- 使用异步框架（FastAPI/Starlette），避免同步阻塞
- 数据库连接池（SQLAlchemy pool_size + max_overflow）
- 缓存策略（Redis 缓存热点数据、响应缓存）
- 限流保护（令牌桶/滑动窗口）
- 无状态设计（JWT Token），方便水平扩展
- 消息队列削峰（Celery/RQ）

### Q10: RESTful API 设计中，如何做版本管理？
**参考答案**：
- URL 版本：`/api/v1/users`（最常见，简单直观）
- Header 版本：`Accept: application/vnd.api+json;version=1`
- Query 参数版本：`/api/users?version=1`
- 推荐 URL 版本 + 语义化版本管理（大版本 URL，小版本兼容）
